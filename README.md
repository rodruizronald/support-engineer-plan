# Support Engineer in 12 Months — Learning Plan

A bottom-up, project-driven path from zero programming experience to a junior support engineer role in SaaS or backend products.

## The 7 Phases

### Phase 1 — Hardware Fundamentals Through Python (Weeks 1–8)

Build the operator's mental model of the machine: CPU, RAM, disk, the operating system as middleman, files and I/O, and processes. Python is learned as a side effect of investigating real hardware behavior. See `phase1/README.md` for the detailed plan.

### Phase 2 — Networking, HTTP, APIs, JSON (Weeks 9–20)

Build deep fluency in the request/response model that powers nearly every production system. Cover IP and DNS, TCP, HTTP/1.1 as plain text, HTTPS and TLS, JSON, REST conventions, the `requests` library, authentication, and the core debugging toolkit (`curl`, `jq`, browser DevTools).

### Phase 3 — Linux, the Shell, and Production Realities (Weeks 21–33)

Move from a single Python script on a laptop to the world of running services and remote machines. Cover Linux fundamentals, the shell beyond basics, SSH, `cron`, the `logging` module, reading stack traces fluently, basic Docker, and writing tools verified with `pytest`.

### Phase 4 — Reading Code and Open Source Contribution (Weeks 34–41)

Practice the skill support engineers actually use most: reading unfamiliar code. Cover Git fluency beyond the basics, navigating projects with `grep` and an editor, writing pull request descriptions, responding to code review, and contributing small real fixes to open source Python projects.

### Phase 5 — Communication and Documentation (Weeks 42–45)

The non-code half of the job. Cover writing clear bug reports, writing runbooks for repeatable procedures, writing post-mortems for incidents, asking questions that get useful answers, and producing documentation that other engineers actually read.

### Phase 6 — Job Application and Portfolio Polish (Weeks 46–49)

Convert the work of the previous phases into a credible application package. Cover writing a technical resume, polishing GitHub repo READMEs, drafting a portfolio page, writing tailored cover letters, and identifying companies that hire junior support engineers.

### Phase 7 — Interview Practice (Weeks 50–52 and ongoing)

Practice explaining technical work out loud. Cover walking through past projects, live HTTP debugging exercises, reading unfamiliar code in interview conditions, common support engineer interview formats, and behavioral question preparation.

## How to Use This Plan

Each phase has its own detailed plan in its own folder (e.g., `phase1/README.md`), expanded as the learner reaches it. This document is only the high-level map.
